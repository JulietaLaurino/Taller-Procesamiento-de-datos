Paso 2: Unir bases de datos

Si bien tengo dos versiones del experimento, quiero tener todos los datos juntos en una nica base de datos. Es por eso que vamos a unir las dos bases de datos una arriba de la otra dado que comparten las mismas variables (columnas). Esto se hace con bind_rows.

Paso 4: Filtro filas

Ahora bien, Gorilla nos da muchiiiisima informacin acerca de cada participante y para analizar los datos es probable que no la necesitemos toda. Un primer paso para el filtrado de toda esta data es quedarme slo con las filas que tienen informacin importante. Para esto uso la funcin filter, la cual extrae (y se queda con) las filas que cumplen con un criterio lgico (el cual nosotres vamos a especificar).

En este caso, me interesan slo las respuestas asociadas al target y eso lo determino quedndome slo con aquellas filas en las que la variable Screen.Name sea target. Adems, me quiero quedar slo con las filas en las que la variable display sea Tarea para no analizar los datos que corresponden al demo inicial.

data_raw <- filter (data_raw, Screen.Name == "target", display == "Tarea")

Fijate que todo esto se indica con el operador relacional == el cual chequea si cada elemento del primer vector es igual al segundo (no confundir con el operador = que se usa para asignar).

Paso 5: Selecciono columnas

Otro paso importante para descartar informacin que no me interesa analizar es la seleccin de columnas. Para esto podemos usar select y especificar con cules queremos quedarnos. Tambin podramos aclarar con qu columnas no queremos quedarnos (es decir, cules queremos que se vayan) agregando un - antes de cada nombre.

data_raw <- select (data_raw, id, group, relation, familiarity, block, prime, target, Response, Acc, RT)

Para analizar RTs: uso mutate para generar una nueva columna que tenga slo los RTs en los que la rta fue correcta.
